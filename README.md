# AMR Insight Platform

A production-style, full-stack antimicrobial resistance and antimicrobial consumption dashboard. The project combines a **Java 21 / Spring Boot backend**, a **PostgreSQL analytical database**, and a **React + TypeScript dashboard**. It also includes Docker, Kubernetes, Argo CD, CI, synthetic demo data, API documentation, and tests.

> The bundled dataset is synthetic and is intended for software demonstrations, portfolio work, and local development. It must not be presented as real patient or hospital data.

## Main capabilities

- Antimicrobial susceptibility dashboard by organism and antibiotic
- Carbapenem-resistance trend by quarter
- Antimicrobial consumption metrics in DDD/100 bed-days
- Top-used antibiotics and Access/Watch grouping
- Specimen and susceptibility result management
- Clinical alerts with severity and workflow status
- Deployment health and synchronization overview
- Jira-style work-item board for delivery tracking
- Read-only dashboard APIs plus authenticated write APIs
- OpenAPI/Swagger documentation
- PostgreSQL Flyway migrations with 5,000 synthetic specimens, 30,000 susceptibility results, and 1,200 monthly consumption records
- Docker Compose local environment
- Kubernetes and Argo CD deployment examples
- GitHub Actions CI pipeline

## Quick start with Docker

```bash
cp .env.example .env
docker compose up --build
```

Open:

- Web dashboard: `http://localhost:3000`
- Backend API: `http://localhost:8080/api`
- Swagger UI: `http://localhost:8080/swagger-ui.html`
- PostgreSQL: `localhost:5432`

Default write credentials are configured through environment variables:

```text
AMR_ADMIN_USERNAME=admin
AMR_ADMIN_PASSWORD=change-me-now
```

Dashboard `GET` endpoints are public in the demo configuration. Creating or updating records requires HTTP Basic authentication.

## Local development

### Backend

Requirements: Java 21 and Maven 3.9+

```bash
cd backend
mvn spring-boot:run
```

### Frontend

Requirements: Node.js 20+

```bash
cd frontend
npm install
npm run dev
```

The Vite development server proxies `/api` to `http://localhost:8080`.

## Useful API routes

| Method | Route | Purpose |
|---|---|---|
| GET | `/api/dashboard/summary` | Headline AMR and consumption indicators |
| GET | `/api/dashboard/susceptibility?organism=Acinetobacter%20species` | Stacked susceptibility data |
| GET | `/api/dashboard/resistance-trend` | Carbapenem-resistance quarterly trend |
| GET | `/api/dashboard/consumption-yearly` | Yearly DDD/100 bed-days |
| GET | `/api/dashboard/top-antibiotics` | Top antibiotic consumption |
| GET | `/api/specimens` | Paginated specimen list |
| POST | `/api/specimens` | Create a specimen (authenticated) |
| GET | `/api/alerts` | List alerts |
| PATCH | `/api/alerts/{id}/status` | Update alert workflow status |
| GET | `/api/deployments` | Runtime and synchronization status |
| GET | `/api/work-items` | Delivery board items |

A ready-to-import Postman collection is available in [`docs/postman/AMR-Insight.postman_collection.json`](docs/postman/AMR-Insight.postman_collection.json).

## Data provenance and privacy

All demo records are generated by SQL using deterministic formulas. Patient references are artificial identifiers such as `P-000001`. No personal data is included. For real deployments, integrate with approved laboratory and pharmacy sources, apply local governance controls, and perform clinical validation before operational use.
